# Security Policy

Thank you for helping keep SpendWise secure. This document explains how to report vulnerabilities.

## Supported Versions

We support the latest release and the previous minor release for critical security fixes. If you're unsure which versions are supported, open an issue.

## Reporting a Vulnerability

Please report security issues privately to the maintainers at security@your-org.example (replace with actual contact) with:
- A clear summary of the issue
- Steps to reproduce
- Impact assessment and mitigation suggestions

Do not include sensitive data in public issue trackers.

## Disclosure Process

We will acknowledge receipt within 3 business days and provide status updates. We aim to release a fix and coordinated disclosure timeline.

## Security Best Practices for Deployments

- Keep dependencies up-to-date and apply security patches regularly.
- Run the application with least privilege principles.
- Use HTTPS/TLS for all network traffic.
- Store secrets in a secure vault (e.g., HashiCorp Vault, cloud KMS) and never commit them to source control.
